## 优化内容

### 1. 请求配置优化

**文件**: `src/services/config.ts`

- 超时时间从 120秒 增加到 **180秒**
- 请求间隔从 300ms 减少到 **100ms**，提高并发能力
- 重试次数从 2次 增加到 **3次**
- 重试间隔从 5000ms 减少到 **3000ms**

### 2. 持久化缓存服务

**文件**: `src/services/persistentCache.ts` (新增)

新增基于 localStorage 的持久化缓存服务：

- **缓存持久化**: 数据缓存到 localStorage，页面刷新后仍然可用
- **版本控制**: 支持缓存版本号，数据格式升级时自动清理旧缓存
- **自动清理**: 缓存过期自动清理，localStorage满时自动删除旧缓存
- **双缓存策略**: 内存缓存 + 持久化缓存，兼顾速度和持久性

**缓存策略**:
| 数据类型 | 内存缓存 | 持久化缓存 |
|---------|---------|-----------|
| 基金列表 | 10分钟 | 30分钟 |
| 基金排行 | 30分钟 | 1小时 |
| 净值估算 | 5分钟 | 不缓存 |

### 3. 基金服务优化

**文件**: `src/services/fundService.ts`

- 优先从内存缓存读取
- 内存缓存未命中时，从持久化缓存读取
- 新增缓存写入时同时更新两个缓存层
- 新增 `getFundRankPaged` 分页获取方法
- **修复API接口名称**: `fund_info_index_em` → `fund_open_fund_rank_em`
- **修复动态字段名处理**: 部分接口返回的字段名包含日期前缀，需要动态匹配
- 优化日志输出，便于调试

**API接口修正**:
根据AKShare官方文档，基金排行正确的接口名称是 `fund_open_fund_rank_em`，参数为 `symbol`（基金类型）。

```python
# AKShare Python示例
import akshare as ak
fund_rank_df = ak.fund_open_fund_rank_em(symbol="全部")
```

**动态字段名处理**:
部分AKShare接口返回的字段名包含具体日期，如 `2026-02-25-单位净值` 而不是简单的 `单位净值`。项目已添加动态字段匹配逻辑来处理这种情况。

| 接口 | 静态字段 | 动态字段示例 |
|------|---------|-------------|
| fund_open_fund_daily_em | 基金代码, 基金简称, 日增长值 | 2026-02-25-单位净值 |
| fund_value_estimation_em | 基金代码, 基金名称, 估算偏差 | 2026-02-26-估算数据-估算值 |

### 4. 数据加载器优化

**文件**: `src/services/dataLoader.ts`

- 使用 `Promise.allSettled` 替代 `Promise.all`，避免单个请求失败影响全部
- 超时时间增加到 180秒
- 添加部分成功处理，即使部分数据加载失败也能显示可用数据
- 缓存检查同时检查持久化缓存

### 5. 搜索对话框优化

**文件**: `src/components/FundSearchDialog.tsx`

- **防抖搜索**: 输入后 300ms 自动搜索，减少频繁请求
- **本地搜索**: 数据加载后在本地进行搜索，无需再次请求API
- **持久化缓存优先**: 打开对话框时优先从持久化缓存加载
- **状态保持**: 关闭对话框后保留已加载的数据，下次打开更快

### 6. 排行列表优化（分页加载）

**文件**: `src/components/FundRankList.tsx`

- **分页加载**: 初始只加载30条，点击"加载更多"每次加载50条
- **类型缓存**: 每种类型独立缓存，切换类型不重新加载
- **优先读缓存**: 优先从持久化缓存加载，避免重复请求
- **后台刷新**: 估算数据后台静默刷新，不阻塞UI

**分页参数**:
```typescript
const INITIAL_SIZE = 30;  // 初始加载数量
const PAGE_SIZE = 50;     // 每次加载更多数量
```

## 性能提升

### 首次加载
- 基金列表: ~30-60秒（取决于网络）
- 基金排行: ~20-40秒（加载全部数据）
- 净值估算: ~15-30秒

### 后续访问（使用持久化缓存）
- 基金列表: **< 100ms**（从localStorage读取）
- 基金排行: **< 100ms**（从localStorage读取）
- 切换排行类型: **< 100ms**（已缓存的类型）
- 净值估算: ~15-30秒（需要实时数据）

### 搜索体验
- 首次搜索: 需要等待数据加载
- 后续搜索: **即时响应**（本地搜索）

### 分页体验
- 初始显示: 30条（快速渲染）
- 加载更多: 每次50条（平滑体验）
- 避免了一次性渲染大量数据导致的卡顿

## 错误处理

### 超时处理
- 请求超时时间增加到180秒
- 超时时显示友好提示："请求超时，AKTools服务器响应较慢，请稍后重试"

### 降级策略
- 部分数据加载失败时，显示可用的数据
- 网络错误时，优先使用缓存数据

## 使用建议

1. **首次使用**: 首次打开应用需要加载全部基金数据，请耐心等待
2. **后续使用**: 数据会自动缓存，下次打开速度会大幅提升
3. **切换类型**: 切换排行类型时，已加载过的类型会立即显示
4. **加载更多**: 点击"加载更多"按钮分批加载数据
5. **手动刷新**: 点击刷新按钮可清除缓存并重新加载最新数据
6. **网络问题**: 如果遇到网络问题，可稍后重试或检查网络连接
