# 修复说明 - 2026 年 3 月 1 日

## 修复内容

### 1. 类型安全问题修复

**问题描述**：代码中存在多处使用 `any` 类型，降低了类型安全性和代码可维护性。

**修复文件**：`src/services/fundManager.ts`

- `NavUpdateEvent.nav` 类型从 `any` 改为 `FundNAV | undefined`
- `fetchLatestNav` 方法中缓存类型从 `any` 改为 `FundNAV`
- `refreshFundNav` 方法中缓存类型从 `any` 改为 `FundNAV`
- `getFundInfo` 方法中内部类型定义使用 `FundNAV` 替代 `any`

**影响范围**：
- 基金净值更新事件类型更加明确
- 缓存数据使用强类型，提高代码可靠性

---

### 2. 性能问题修复 - addFund

**问题描述**：`addFund` 方法每次都调用 `getOpenFundList()` 获取全量基金数据，效率低下。

**修复方案**：
- 在 `fundService.ts` 中新增 `getFundByCode(code: string)` 方法
- 该方法优先从缓存获取单只基金信息
- 避免每次添加基金都请求全量数据

**修复文件**：
- `src/services/fundService.ts` - 新增 `getFundByCode` 方法
- `src/services/fundManager.ts` - 修改 `getFundInfo` 调用

**性能提升**：
- 添加单只基金时，减少不必要的全量数据请求
- 单只基金信息缓存 5 分钟

---

### 3. 性能问题修复 - refreshAllNavs

**问题描述**：`refreshAllNavs` 方法使用串行方式刷新基金净值，效率低下。

**修复前**：
```typescript
for (const fund of funds) {
  const updated = await this.refreshFundNav(fund.code);
  // ...
}
```

**修复后**：
```typescript
const results = await Promise.allSettled(
  funds.map(fund => this.refreshFundNav(fund.code))
);
```

**修复文件**：`src/services/fundManager.ts`

**性能提升**：
- 从串行改为并发刷新
- 多只基金净值同时更新，响应时间大幅缩短

---

### 4. 错误处理增强

**问题描述**：API 请求失败时直接抛出异常，没有降级处理。

**修复方案**：在以下方法中增加过期缓存降级机制：

- `getFundRank` - 失败时返回过期缓存
- `getFundEstimate` - 失败时返回过期缓存

**修复文件**：`src/services/fundService.ts`

**用户体验**：
- 即使 API 请求失败，也能展示过期数据
- 避免用户看到空白页面
